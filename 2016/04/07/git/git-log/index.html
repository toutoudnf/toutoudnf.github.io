Preface

git log太牛逼了，下面简单列下几种情况，方便复习。

详情点此连接

Cases & Point

基本点

  • log 中的sha-1，长度是不固定的，只要在记录中能唯一就ok
    • 测试时，最短是4位
  • git reflog查看本地HEAD引用的历史记录
  • git log –abbrev-commit 可以查看sha-1的简易值
  • git rev-parse master，查看master指向的commit

关于commit范围查找

  • 主要是方便定位不同分支之间的commit差异
    • 其中的~^这种方式,对commit来讲是通用的
  • 在commit尾部加^，代表父commit
  • 在commit尾部加^2，代表第二父commit
  • 在commit尾部加~，代表父commit，等价于^
  • 在commit尾部加~2，代表父commit的父commit
  • HEAD~3^2，代表HEAD的父-父-父commit的第二父commit（如果有）
  • HEAD^^^^等价于HEAD~4
  • 双点
    • master..develop
      • 不包含master但是包含develop的commit
    • master..
      • 等价于master..HEAD
  • 在commit前加^,表示不包含该commit;分之前则不包含该分支的commit
    • 如:git log master..develop等价于git log ^master develop
    • 还,等价于git log develop –not master
  • 多点
    • git log master…develop
      • master和develop中，除了共通之外的commit

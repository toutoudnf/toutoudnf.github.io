hexo搭建和备份记录

发表于 2016-12-06 | 分类于 hexo

背景

换过几次环境，也尝试过几种备份方式，现在将过程&碰到的坑记录如下，以便后续查找&参考。

现状

  • 目前source是备份在github上的，只备份了_post下的源文件。之前备份的整个hexo目录，但是在windows下clone下来并没有办法直接使用，故替换了备份方案。
  • 目前使用的css是next，简单简洁，地址是：
    • next on github

准备

  • hexo官网
  • 参考官网步骤即可
  • 简单说下
    • 安装nodejs（可通过nvm）
    • 安完nodejs就有了npm
    • 用npm安装hexo
    • 然后在hexo下clone next，然后配置

碰到的坑

tags和categories未生成

  • 目前发现的问题是tags下的index.md中缺少type: "tags"这个配置
  • 详情可以参考这个知乎问题

tags和categories一点击，http变https

  • 临时修复方案是在next的_config.yml中写死https的路径，坏处是在本地没法玩了
  • 目前新版本的next貌似没遇到这个问题
  • 具体可关注issues
    • next issues

hexo在本地启动，无法访问

通常来讲是默认4000端口被占用了，可以：

  • windows下
    • netstat -ano|findstr 4000查看pid
    • tasklist|findstr pid查看具体进程

碰到的是foxitProtect，taskmgr干掉即可。

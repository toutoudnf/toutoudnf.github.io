jvm学习思路分析.md

发表于 2017-07-06 | 分类于 java

梳理下 JVM 的学习思路。也算是对自己 JVM 学习路径的一个记录和总结，方便分享给其他人。

学习一个东西，带着目的去学，是比较高效的。在开始系统学习之前，先梳理下个人学习 JVM 的目标。

相信跟我一样对 JVM 一知半解的 java dev 还是比较多的，各个零散的渠道获得了不少信息，但也不知道是对是错。如果能通过这个系列帮到相关的 java dev，也算是功德一件：）

思路梳理

总体来说，是对自己技术体系 + 面试有效的，细分项是：

  • 对日常编程开发的指导，知道细节，能规避那些坑
  • java app 出现性能问题的时候，如何优化
  • 是 VM 的一种，对一些操作系统基础知识的熟悉
  • 作为 java server end R&D，一个不错的加分项

能想到的相关途径

  • 相关 tool 或者监控工具的熟悉
  • JVM 参数的学习
  • JVM 原理，包括架构，细节的学习
  • 问题场景的收集，mock，然后通过学习的知识进行调优（实战）
  • 搜搜面试题，关注并统计面试的时候涉及到的是什么

现在有的资源

  • oracle官网
  • 书籍
    • Java 性能优化权威指南
    • 深入理解 Java 虚拟机
  • 持续补充

我要达成的目标

  • 建立 JVM 的学习路径，并依次学习。脑袋中对 JVM 有个完整的框架
    • JVM 的架构
      • 以 HotSpot 为例，由哪几个组件组成，分别的职责是什么
      • 早期的 JVM 就是这个结构么？在发展过程中，什么阶段加入了什么，是采用什么思路，解决了什么问题？
      • 如果需要，需要对组件进行细分，比如垃圾回收下面可能包括子组件等类似情况
    • VM 进阶
      • 这部分是要明白 VM 的工作都干了啥，即 VM 这个整体的职责。在我看来就是 VM 怎么初始化的，截至到 VM 能对外提供服务，比如执行代码； VM 初始化完成以后，如何对外提供能力；VM是如何销毁的；以及我们应该如何知道 VM 是正常工作的。
      • VM 初始化，在这里应该知道 VM 在初始化的时候都做了些什么事儿
        • 参数
          • 常用启动参数，及其作用
          • 各个版本的参数查询，以及如何看懂参数列表
        • 类加载
          • 类初始化的过程
          • 类的生命周期
        • 补充ing，看看还有啥
      • VM 运行时
        • 类加载
          • 这里的类加载关注的是加载器的加载顺序等，跟应用类相关的
        • 内存分配
          • 创建新的对象的过程 + 策略（在什么区创建）
        • 垃圾回收
          • 对象无用的判断
          • 对象回收gc，下面的项，加起来就是gc的最基本的工作原理
            • 谁触发的 gc？
            • 如何控制gc（参数）
            • gc 的种类
            • gc 的工作步骤
            • gc 的策略（common策略 + 每种单独的策略）
        • 异常处理
          • VM 初始化出现了异常
          • 类执行出现了异常
      • 其他内部机制
        • JIT
        • 优化
          • 编译期优化
          • 运行期优化
          • 安全与锁优化
          • 并发线程优化
        • TODO 待补充
      • VM 的监控
        • 工具
        • 哪些基本的参数值得关注
  • mock 常见问题场景，并总结自己的定位 + 优化思路
    • 熟悉 jvm 常见 tool，如 jdk 中的那些
    • 收集 jvm 常见的/可能的异常场景
      • 如果可以，尝试通过代码 + 配置来复现场景，然后分析
      • 对场景分类并总结，列举一些常见的定位和调优方式
    • 企业级/应用级的 jvm 监控方案
  • 知道如何查找，并关注 JVM 接下来的 release 和发展
  • 知道如何关注其他 VM 模式的语言的特性和发展，相关信息，如 python
  • 完成自己编写简单 JVM
  • 如果学习笔记整理的 nice，尝试参考 程序猿投稿分析，并发表相关的博客，混脸熟，耶

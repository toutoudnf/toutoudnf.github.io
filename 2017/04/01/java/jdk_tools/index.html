jdk tools

jps

  • Java Virtual Machine Process Status Tool
  • 看local jvm（支持instruments）的tool
    • remote那个没看懂
  • 参数
    • -m可以看command的参数
    • -l可以看start jar or class的完整名称
    • -v可以看-D的参数

jstack

jstat/jstatd

  • Java Virtual Machine Statistics Monitoring Tool
  • 显示了jvm的gc相关的信息
  • 可以参考jvm command的系列
  • jstatd用作deamon进程，可以作为remote monitor jvm的一种方式，类似jmx
  • 看之前，要知晓
    • eden ==》 survivor ==》 old 的过程
    • permGen是什么
  • TODO：
    • jstatd通过visualvm的远程连接没成功

common部分的一些说明：

jstat -help|-options # 看哪些options可用
jstat -<option> [-t] [-h<lines>] <vmid> [<interval> [<count>]]

针对：

  • t：每行加时间戳
  • h：隔x行就re-output一遍header
  • interval：间隔时间，支持两种格式：5s/500ms；默认类似-1的效果，值需>0
  • count：重复几次；默认no limit

下面是临时的笔记

jconsole

  • 根据文章，完成了jconsole的远程连接配置：戳这里

catalina.sh中增加的配置：

JAVA_OPTS="$JAVA_OPTS -Dcom.sun.management.jmxremote.port=11111"
JAVA_OPTS="$JAVA_OPTS -Dcom.sun.management.jmxremote.authenticate=false"
JAVA_OPTS="$JAVA_OPTS -Dcom.sun.management.jmxremote.ssl=false"
JAVA_OPTS="$JAVA_OPTS -Dcom.sun.management.jmxremote.pwd.file=/usr/lib/jvm/java-1.7.0-openjdk-1.7.0.99-2.6.5.0.el7_2.x86_64/jre/lib/management/jmxremote.password"

并且测试了tomcat stop时，没有报端口被占用的问题

其中：

  • com.sun.management.jmxremote.authenticate
    • true表示使用验证
    • false表示不使用
  • com.sun.management.jmxremote.pwd.file
    • 当auth被set 为true的时候需要

agent端给的参数是：

java -Djava.ext.dirs=./icsAgent_lib/ -Dcom.sun.management.jmxremote.port=22222 -Dcom.sun.management.jmxremote.authenticate=false -Dcom.sun.management.jmxremote.ssl=false

testNG on jenkins

发表于 2017-01-24 | 分类于 test

测试用例与自动化工具jenkins的结合，也是个值得记录和分析的地方。

测试用例结合了自动化部署&测试环境后（本文是jenkins），使用的过程中会存在一些痛点，记录并分析。

诉求

截止到目前，对测试用例的一些需求，简单总结一下

  • console log
  • testNG report
  • 用例在CI的小范围运行测试
    • 问题复现
    • 快速测试

分开来说说

console log

log其实包含两方面：

  • 业务log
  • jenkins job log
    • job每个step的log
    • testNG log

常见的问题通常是：

  • 出了bug最好能在testNG console log就定位出来问题
  • testNG console log搞不定，最好能给一个tips，方便在业务的log中定位问题
    • log 时间
    • task id

如果更牛逼的是，能通过一个唯一的identifier，来grep出一串log就更好了。当然这大概是日志收集系统的一部分工作了，暂且不表。

log大概是这些

testNG report

靠人去统计真的是不现实的，因为：

  • 构建频繁之后，累死你
  • 容易出错
  • 还浪费时间，累且没人爱干

所以思路大概是：

  • 自己通过testNG钩子实现需求
  • 找插件

在搞之前，分析了一下大概的需求有：

  • 时间分析
    • 总时间是多少
    • 每个用例的时间是多少
    • common part时间是多少
    • 是否用例时间发生了很大的变化
  • 结果统计
    • 成功失败数量统计
    • 每个class中，成功失败的数量change
    • best wishes是能关联上author，不过不太好搞。

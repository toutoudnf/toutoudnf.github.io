virtualBox配置的一些心得

发表于 2017-06-29 | 分类于 virtualization

记录了使用virtualBox 过程中碰到的一些问题，方便日后查询。

配置 NAT network

  • 虚拟机之间互通
  • 主机与 VM 之间默认不能通讯

配置的位置在：

virtual菜单==》偏好设置==》网络==》NAT网络，新建一个NAT网络并在创建虚拟机的时候指定。使用 DHCP 方式即可自动获取 IP。

NAT 网络模式下，HOST ssh to VM

  1. 在 virtual菜单==》偏好设置==》网络==》NAT网络 中，选择编辑 NAT 网络==》端口转发，增加一条转发规则为（假设 VM ip 为 10.0.2.5）：Rule名称，TCP，127.0.0.1，22222，10.0.2.5，22。这样就是配置了主机22222端口将收到的 TCP 请求都转发到 10.0.2.5 的 22 端口上。
  2. 确认 VM 的 sshd 开启，并且配置了 iptables，如：
    1. service sshd start
    2. service iptables stop

之后在主机上通过 ssh -p 22222 root@127.0.0.1，即可完成到 VM 的连接。

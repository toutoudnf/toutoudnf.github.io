git log

Preface

git log太牛逼了，下面简单列下几种情况，方便复习。

详情点此连接

Cases & Point

基本点

  • log 中的sha-1，长度是不固定的，只要在记录中能唯一就ok
    • 测试时，最短是4位
  • git reflog查看本地HEAD引用的历史记录
  • git log –abbrev-commit 可以查看sha-1的简易值
  • git rev-parse master，查看master指向的commit

关于commit范围查找

  • 主要是方便定位不同分支之间的commit差异
    • 其中的~^这种方式,对commit来讲是通用的
  • 在commit尾部加^，代表父commit
  • 在commit尾部加^2，代表第二父commit
  • 在commit尾部加~，代表父commit，等价于^
  • 在commit尾部加~2，代表父commit的父commit
  • HEAD~3^2，代表HEAD的父-父-父commit的第二父commit（如果有）
  • HEAD^^^^等价于HEAD~4
  • 双点
    • master..develop
      • 不包含master但是包含develop的commit
    • master..
      • 等价于master..HEAD
  • 在commit前加^,表示不包含该commit;分之前则不包含该分支的commit
    • 如:git log master..develop等价于git log ^master develop
    • 还,等价于git log develop –not master
  • 多点
    • git log master…develop
      • master和develop中，除了共通之外的commit

实际应用场景

查询指定commit的变更详情

  • 详情一般包括一下几点：
    • 指定commit都改了哪些文件
      • –stat是看文件
    • 改动详情是什么
      • 其实这个可以用diff做
      • -p是看细节

具体如下：

  • git log –stat
    • 可查询commit的文件变更情况
    • 结果如下

查询commit的commiter

  • 碰上这个问题是rebase之后，以为修改author没修改committer导致git review提不上去
  • 查询方式就是git log –format=full
    • 此时的log信息中就包含了commiter

git常见问题记录

发表于 2016-04-05 | 分类于 git

这里记录了git使用过程中碰到的问题.
同时也记录了一些小tips

Questions

Unlink of file failed

  • pull branch occurs
  • show the unlink of files, ask to try again or not
  • the reason is sth file is used by other programs
  • could be resolved by:
    • git gc
    • close all the program that using the git file
  • reference url
    • https://stackoverflow.com/questions/10181057/unlink-of-file-failed

git LF warning

碰到的问题是提交代码的时候一大堆的warning，很不爽

  • stackoverflow有个说明非常棒，戳这里

简单讲自己理解

  • warning是对core.autocrlf当前值的警告
  • core.autocrlf控制了文件的push & fetch过程中，git是否会对换行符做处理
  • 具体有个图可以参考上面的答案，很直观

windows对大小写不敏感的问题

修改git大小写敏感的配置即可

git config core.ignorecase false

想要修改author或者committer

查询当前commit的author和committer信息

git show commit_hash –pretty=full –stat

github去计算contribute的时候，是按照邮箱来的，所以主要确认下邮箱是否正确

然后如果有问题就改咯

git commit –amend –author=”toutoudnf toutoudnf@gmail.com“

就酱，改完收工
